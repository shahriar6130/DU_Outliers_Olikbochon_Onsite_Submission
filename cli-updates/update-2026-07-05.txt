WHY v1-v8 PLATEAUED AT ~0.65
46% of the test set is [NULL] context (closed-book). BanglaBERT/NLI/e5/hand
features only measure response<->context consistency -> near-random on closed-book
-> hard ceiling ~0.65. Confirmed: a consistency-only mock scores 0.658 on the 299.

WHAT v10 DOES (breaks the ceiling with world knowledge; APIs are banned so it's local)
1. Two-regime routing: context rows vs closed-book rows judged differently.
2. LLM judge (vLLM): single-token Yes/No logprob for context rows;
   chain-of-thought + self-consistency (k=5 majority vote) for closed-book rows.
3. RAG over Bengali Wikipedia: retrieve evidence for closed-book rows, re-judge
   with the passage (auto-skips if no corpus attached).
4. One regularized threshold per regime (dropped the overfit 30x30 grid).
5. Dropped pseudo-labeling (it crashed v3 to 0.466) and the heavy ML stack.
   Judge-dominant; NLI/e5/LR available as an optional low-weight tie-break.
