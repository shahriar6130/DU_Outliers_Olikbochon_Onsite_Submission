PROBLEMS FIXED (original v3.0):
1. LayerNorm key mismatch (gamma/beta -> weight/bias): BanglaBERT checkpoint uses old
   PyTorch naming, but ElectraForSequenceClassification expects weight/bias. ALL LayerNorm
   layers were randomly initialized. Fixed via create_banglabert_classifier() which loads
   base ElectraModel and manually remaps state_dict keys.
2. Only 1 fold training: original trained on fold 0 only (waste of data). Fixed to 5-fold CV.
3. No class imbalance handling: added WeightedTrainer with class weights.
4. No NLI integration: added zero-shot NLI entailment scores as auxiliary features.

NEW STRATEGIES ADDED (v3.0 enhanced):
1. Data Augmentation (EDA): character-level random swaps/deletions/insertions on Bangla text.
   Generates 3x augmented copies per sample (299 -> 1196 total training rows).

2. 5-Fold Fine-Tuning Ensemble: fine-tune BanglaBERT on each fold, collect OOF predictions.
   Uses EarlyStopping (patience=3) with Macro F1 as the tracking metric.

3. Feature Extraction + XGBoost Meta-Classifier:
   - Freeze the fine-tuned BanglaBERT, extract [CLS] embeddings (768-dim)
   - Concatenate with NLI scores -> 769-dim feature vectors
   - Train XGBoost with 5-fold CV on these features
   - XGBoost handles small data better than fine-tuning the full transformer

4. Blended Ensemble: search optimal weights (0.0-1.0) for combining:
   - Fine-tuned probabilities
   - XGBoost probabilities
   - NLI raw scores
   Weights found via grid search on OOF predictions maximizing Macro F1.

5. Pseudo-Labeling (key for 0.8 target):
   - Predict test set (2516 samples) with blended ensemble
   - Select samples with confidence > 0.8 (|prob - 0.5| * 2 > 0.8)
   - Add pseudo-labeled samples to training set
   - Retrain XGBoost on expanded set
   - This effectively increases training data 3-8x

6. Test-Time Augmentation (TTA):
   - Generate 5 noisy variants of each test sample using same EDA
   - Average predictions across all variants for smoother, more robust output

MODEL PIPELINE (execution order):
1. Load & normalize data (BUET normalizer)
2. Compute NLI entailment scores for all data (mDeBERTa-v3 zero-shot)
3. Generate augmented training data (3x EDA)
4. 5-fold BanglaBERT fine-tuning -> OOF predictions
5. Extract [CLS] embeddings from fine-tuned model
6. Train XGBoost on embeddings + NLI scores (5-fold CV)
7. Blend FT + XGB + NLI (grid search weights on OOF)
8. Pseudo-label test set high-confidence samples
9. Retrain XGBoost on expanded set
10. TTA on test set (5 augmented variants averaged)
11. Generate final submission.csv
