PROBLEMS FIXED (original v3.0):
1. LayerNorm key mismatch (gamma/beta -> weight/bias): BanglaBERT checkpoint uses old
   PyTorch naming, but ElectraForSequenceClassification expects weight/bias. ALL LayerNorm
   layers were randomly initialized. Fixed via create_banglabert_classifier() which loads
   base ElectraModel and manually remaps state_dict keys.
2. Only 1 fold training: original trained on fold 0 only (waste of data). Fixed to 5-fold CV.
3. No class imbalance handling: added WeightedTrainer with class weights.
4. No NLI integration: added zero-shot NLI entailment scores as auxiliary features.

NEW STRATEGIES ADDED (v3.0 enhanced):
1. Data Augmentation (EDA): character-level random swaps/deletions/insertions on Bangla text.
   Generates 3x augmented copies per sample (299 -> 1196 total training rows).

2. 5-Fold Fine-Tuning Ensemble: fine-tune BanglaBERT on each fold, collect OOF predictions.
   Uses EarlyStopping (patience=3) with Macro F1 as the tracking metric.

3. Feature Extraction + XGBoost Meta-Classifier:
   - Freeze the fine-tuned BanglaBERT, extract [CLS] embeddings (768-dim)
   - Concatenate with NLI scores -> 769-dim feature vectors
   - Train XGBoost with 5-fold CV on these features
   - XGBoost handles small data better than fine-tuning the full transformer

4. Blended Ensemble: search optimal weights (0.0-1.0) for combining:
   - Fine-tuned probabilities
   - XGBoost probabilities
   - NLI raw scores
   Weights found via grid search on OOF predictions maximizing Macro F1.

5. Pseudo-Labeling (key for 0.8 target):
   - Predict test set (2516 samples) with blended ensemble
   - Select samples with confidence > 0.8 (|prob - 0.5| * 2 > 0.8)
   - Add pseudo-labeled samples to training set
   - Retrain XGBoost on expanded set
   - This effectively increases training data 3-8x

6. Test-Time Augmentation (TTA):
   - Generate 5 noisy variants of each test sample using same EDA
   - Average predictions across all variants for smoother, more robust output

MODEL PIPELINE (execution order):
1. Load & normalize data (BUET normalizer)
2. Compute NLI entailment scores for all data (mDeBERTa-v3 zero-shot)
3. Generate augmented training data (3x EDA)
4. 5-fold BanglaBERT fine-tuning -> OOF predictions
5. Extract [CLS] embeddings from fine-tuned model
6. Train XGBoost on embeddings + NLI scores (5-fold CV)
7. Blend FT + XGB + NLI (grid search weights on OOF)
8. Pseudo-label test set high-confidence samples
9. Retrain XGBoost on expanded set
10. TTA on test set (5 augmented variants averaged)
11. Generate final submission.csv

Update: 2026-07-02 — v4.0 Implementation (Feature Engineering + Light Classifier)
================================================================================

NOTEBOOK: myNotebooks/model_v4.0.ipynb
OUTPUT: /kaggle/working/submission_v4.0.csv

STRATEGY: No fine-tuning. Use pretrained models as feature extractors only.
With only 299 training samples, fine-tuning large transformers (v3.0) overfits.
Instead, stack pretrained signals with hand-crafted features and a light classifier.

WHY v3.0 FAILED (score dropped from 0.568 to 0.466):
1. BanglaBERT manual weight loading likely corrupted backbone weights
   - ElectraForSequenceClassification constructed from config, weights mapped manually
   - Subtle LayerNorm key mismatches silently broke the model
2. Input format wrong: "context [SEP] prompt [SEP] response" as single string
   - BERT expects two sequences: [CLS] A [SEP] B [SEP], not one long string
   - MAX_LEN=256 truncates long contexts, losing critical info
3. XGBoost trained on fold-0 embeddings only, not proper ensemble
4. Grid search on 299-sample OOF overfits to training distribution
5. n0/n1 recomputed at test time (bug in line 409)

WHY v2.0 WORKED (score 0.568):
1. Used NLI model as-is (no fine-tuning risk)
2. Smart context/no-context split: NLI for context rows, TF-IDF for closed-book
3. Simple threshold (0.5) on NLI entailment score is principled

v4.0 PIPELINE:
1. Hand-crafted features (20+ features):
   - Text length: context_len, prompt_len, response_len, ratios
   - Token overlap: Jaccard similarity, common token ratio (ctx-resp, resp-prompt)
   - Number extraction: Bengali + Arabic numeral detection, number matching
   - Structural: resp_very_short, resp_one_word, resp_starts_with_ctx
   - Has context flag

2. NLI dual scores (mDeBERTa-v3 zero-shot, multi_label=True):
   - Entailment: premise=context (or prompt), hypothesis=response
   - Contradiction: premise=response, hypothesis=context (reversed)
   - With context rows only for premise selection

3. Sentence embedding similarity (paraphrase-multilingual-MiniLM-L12-v2):
   - Cosine similarity: context-response
   - Cosine similarity: prompt-response
   - Normalized embeddings for efficient dot-product

4. Light classifier ensemble:
   - LogisticRegression (StandardScaler + balanced class weights)
   - XGBoost (max_depth=4, conservative regularization)
   - Both trained with 5-fold StratifiedKFold CV

5. Blend + threshold optimization:
   - Grid search over 4-component weights: LR, XGB, NLI, SIM
   - Threshold optimization on OOF blend scores
   - Applied to test set predictions

KEY DESIGN DECISIONS:
- NO fine-tuning on 299 samples (avoids v3.0's core failure)
- Bengali numeral extraction (০-৯ mapped to 0-9) for numeric fact checking
- Tokenize via simple whitespace split (no external Bangla tokenizer needed)
- StandardScaler for LR (features have different scales)
- Conservative XGBoost (max_depth=4, min_child_weight=3) to prevent overfitting
- multi_label=True in NLI (matches v2.0's approach that worked)

REQUIRED PACKAGES:
- transformers (NLI pipeline)
- sentence-transformers (embedding similarity)
- xgboost (meta-classifier)
- csebuetnlp/normalizer (Bangla text normalization)
- scikit-learn (LogisticRegression, StandardScaler, StratifiedKFold)

Update: 2026-07-02 — v5.0 Implementation (Multi-Model Feature Engineering + Extrinsic Hallucination)
================================================================================

NOTEBOOK: myNotebooks/model_v5.0.ipynb
OUTPUT: /kaggle/working/submission_v5.0.csv

STRATEGY: Multi-model feature engineering with focus on extrinsic hallucination detection.
Enhances v4.0 with advanced models and richer features.

MODELS USED:
1. MoritzLaurer/mDeBERTa-v3-base-xnli-multilingual-nli-2mil7 (XNLI-BN)
   - Cross-lingual NLI for Bengali
   - Entailment and contradiction scores

2. ntfloat/multilingual-e5-base (Sentence Embeddings)
   - Advanced multilingual sentence embeddings
   - Better semantic similarity than paraphrase-multilingual-MiniLM-L12-v2

3. Davlan/xlm-roberta-base-ner-hrl (Named Entity Recognition)
   - Extract entities from context, prompt, and response
   - Entity overlap and containment features

4. facebook/nllb-200-distilled-600M (Translation)
   - Placeholder for translation features
   - Can be enabled for cross-lingual verification

5. Qwen/Qwen2.5-7B-Instruct (Zero-shot Fact-checker)
   - 4-bit quantized for efficiency
   - Placeholder for factual consistency features

FEATURE ENGINEERING (40+ features):
1. Hand-crafted features (30+):
   - Text length features and ratios
   - Token overlap (Jaccard, common ratio)
   - Number extraction and matching (Bengali + Arabic)
   - Extrinsic hallucination features (words not in context)
   - Factual consistency features (entity overlap)
   - Context completeness features

2. NLI dual scores (XNLI-BN):
   - Entailment: premise=context, hypothesis=response
   - Contradiction: premise=response, hypothesis=context

3. Sentence embedding similarity (multilingual-e5-base):
   - Context-response similarity
   - Prompt-response similarity

4. NER features:
   - Entity counts (context, prompt, response)
   - Entity overlap features
   - Entity containment features
   - Extrinsic entity features (entities not in context)

5. Cross-lingual disagreement features:
   - NLI vs Embedding disagreement
   - Confidence disagreement
   - Consensus features
   - Score variance across models

CLASSIFIER ENSEMBLE:
- LogisticRegression (StandardScaler + balanced class weights)
- LinearRegression (added to ensemble)
- XGBoost (max_depth=4, conservative regularization)
- All trained with 5-fold StratifiedKFold CV

BLEND + THRESHOLD:
- Grid search over 5-component weights: LR, LinReg, XGB, NLI, SIM
- Threshold optimization on OOF blend scores
- Applied to test set predictions

KEY DESIGN DECISIONS:
- NO fine-tuning on 299 samples (avoids overfitting)
- Multi-model ensemble for robust predictions
- Focus on extrinsic hallucination (hallucination where response contradicts or adds to context)
- Cross-lingual NLI (XNLI-BN) for better Bengali understanding
- Proper OOF evaluation with 5-fold CV
- Placeholder for advanced models (Qwen, NLLB) ready to enable

EXTRINSIC HALLUCINATION FEATURES:
- Words in response not in context (count and ratio)
- Entities in response not in context (count and ratio)
- Number matching (response numbers in context)
- Token overlap between response and context

FACTS-BASED CONTEXT FEATURES:
- Numeric fact verification (number extraction and matching)
- Entity fact verification (entity overlap and containment)
- Context completeness (how much context addresses the prompt)
- Factual consistency (NLI + embedding similarity)
